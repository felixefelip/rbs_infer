# Task: gerador RBS para `mount_uploader` (CarrierWave)

Adicionar uma extensão `RbsInfer::Extensions::CarrierWave::Generator` análoga ao gerador de enumerize, que emite RBS sobrescrevendo os acessores de coluna que o `rbs_rails` cria, mais os métodos auxiliares injetados por `mount_uploader` / `mount_uploaders`.

A motivação concreta vem do plano [`carrierwave_user_avatar_upload.md`](carrierwave_user_avatar_upload.md): com a RBS de `CarrierWave::*` resolvida via `gem_rbs_collection`, o `steep check` no dummy passa a falhar em:

```
app/views/users/show.html.erb:4:29: [error] Type `(::String | nil)` does not have method `url`
app/views/users/avatars/edit.html.erb:17:33: [error] Type `(::String | nil)` does not have method `url`
```

A causa é o `rbs_rails` gerar `def avatar: () -> ::String?` a partir da coluna `users.avatar:string`, ignorando que `mount_uploader :avatar, AvatarUploader` reescreve o acessor em runtime.

---

## Estratégia escolhida: pós-processador sobre `sig/rbs_rails/`

A opção implementada deste plano é a **abordagem 2** discutida no chat: reescrever os arquivos `sig/rbs_rails/app/models/<model>.rbs` removendo os defs de coluna conflitantes **antes** de gravar o RBS do gerador de carrierwave. A justificativa:

- Evita patch no `rbs_rails` (que é um repo paralelo — `/Users/felipepessoa/workspaces/rbs_rails`); mantém a mudança autocontida em `rbs_infer`.
- O conflito é só de método (mesma assinatura aparecendo em dois arquivos `sig/`); resolvê-lo de um lado só basta.
- `sig/rbs_rails/` é regenerado por `rake rbs_rails:all`, então o pós-processamento precisa rodar **depois** dele. Isso é fácil de orquestrar via Makefile / rake.

### Trade-off explícito

A reescrita do arquivo do `rbs_rails` é frágil: depende do formato do arquivo e da ordem de execução. Se o usuário rodar `rake rbs_rails:all` sem rodar o `rbs_infer:carrierwave:all` logo em seguida, os arquivos voltam ao estado conflitante e o `steep check` regride. Mitigações:

- Documentar no README do dummy a ordem `rbs_rails → rbs_carrierwave`.
- Adicionar um alvo `rbs_generators_all` no `Makefile` que encadeia os dois.
- (Idealmente, futuramente) trocar pela **abordagem 1** (patch no `rbs_rails` para pular colunas com uploader). Esta task fica como caminho pragmático imediato.

## Detecção de `mount_uploader` no source

Via Prism, mesmo padrão do enumerize: visitar nós do modelo procurando `CallNode` com `name == :mount_uploader` (ou `:mount_uploaders`) e `receiver == nil`.

Argumentos a capturar:
1. **Nome do atributo** — primeiro `SymbolNode` (ex.: `:avatar`).
2. **Classe do uploader** — segundo argumento, normalmente `ConstantReadNode` (`AvatarUploader`) ou `ConstantPathNode` (`User::AvatarUploader`). Resolver via `constant_path_to_string` (já existe no enumerize generator, dá pra fatorar pra um helper compartilhado).
3. **Options** — `KeywordHashNode` opcional: por enquanto ignorar; futuras versões podem aceitar `mount_on:`, `validate_integrity:`, etc.

`mount_uploaders` (plural) usa o mesmo formato mas retorna `Array[UploaderClass]` em vez de `UploaderClass`.

## RBS gerado por uploader montado

Para `mount_uploader :avatar, AvatarUploader` em `User`:

```rbs
# Generated by rbs_infer (carrierwave)
class ::User
  def avatar: () -> ::AvatarUploader
  def avatar=: (untyped) -> untyped
  def avatar?: () -> bool

  def remove_avatar: () -> bool
  def remove_avatar=: (bool | ::String) -> (bool | ::String)
  def remove_avatar?: () -> bool

  def avatar_cache: () -> ::String?
  def avatar_cache=: (::String?) -> ::String?

  def remote_avatar_url: () -> ::String?
  def remote_avatar_url=: (::String?) -> ::String?

  def write_avatar_identifier: () -> void
  def store_avatar!: () -> void
  def remove_avatar!: () -> void
  def store_previous_changes_for_avatar: () -> void
end
```

Para `mount_uploaders :photos, PhotoUploader` (plural) trocar:
- `def photos: () -> Array[::PhotoUploader]`
- `def photos=: (untyped) -> untyped` (CarrierWave aceita `Array[IO | UploadedFile | String]`; deixar `untyped` por enquanto)
- demais auxiliares com nome no plural

### Por que `untyped` em `=`?

Os setters da CarrierWave aceitam uma lista heterogênea (`IO`, `ActionDispatch::Http::UploadedFile`, `String` para cache, `nil`). O fork do `gem_rbs_collection` já tem tipos para a maioria desses inputs, mas amarrar agora pode gerar falsos positivos com formulários. Começar com `untyped` e refinar depois.

## Pós-processamento de `sig/rbs_rails/app/models/<model>.rbs`

Depois de gerar o RBS do uploader, abrir o arquivo do `rbs_rails` correspondente ao mesmo modelo e remover os defs da(s) coluna(s) montada(s). O arquivo do rbs_rails para `User` tem (resumido):

```rbs
class User
  module GeneratedAttributeMethods
    def avatar: () -> ::String?
    def avatar=: (::String?) -> ::String?
    def avatar?: () -> bool
    def avatar_changed?: ...
    def avatar_change: ...
    # ... mais ~15 métodos da AR
  end
end
```

Estratégia:

1. **Ler** `sig/rbs_rails/app/models/<model>.rbs`.
2. **Identificar os blocos** correspondentes ao atributo montado. Cada bloco é um `def <attr>...: ...` linha única. Usar regex ancorado em `^( *)def <attr>(?:_[a-z_]*|=|\?)?: .*$` é suficiente — formato do rbs_rails é estável (uma assinatura por linha, sem overloads).
3. **Remover** os defs que conflitariam com o RBS do gerador. Lista mínima a remover (para um attr `avatar`):
   - `def avatar`
   - `def avatar=`
   - `def avatar?`
   - `def avatar_changed?`, `def avatar_change`, `def avatar_will_change!`, `def avatar_was`, `def avatar_previously_changed?`, `def avatar_previous_change`, `def avatar_previously_was`, `def avatar_before_last_save`, `def avatar_change_to_be_saved`, `def avatar_in_database`, `def saved_change_to_avatar`, `def saved_change_to_avatar?`, `def will_save_change_to_avatar?`, `def restore_avatar!`, `def clear_avatar_change`, `def avatar_before_type_cast`, `def avatar_for_database`

   Decisão: **remover só** os 3 acessores básicos (`avatar`, `avatar=`, `avatar?`). Os `*_changed?` / `*_was` / `*_before_type_cast` ainda retornam `String?` em runtime (o que está em DB), então o type do rbs_rails está correto pra eles; manter.

4. **Reescrever** o arquivo. Idempotente: rerodar não causa diff.

### Esqueleto da remoção

```ruby
def strip_column_methods!(rbs_path, attr_name)
  return unless File.exist?(rbs_path)

  pattern = /^\s*def #{Regexp.escape(attr_name)}(?:=|\?)?:.*\n/
  content = File.read(rbs_path)
  stripped = content.gsub(pattern, "")
  File.write(rbs_path, stripped) if stripped != content
end
```

## Rake task

`lib/rbs_infer/extensions/carrierwave/tasks/rbs_infer_carrierwave.rake`:

```ruby
namespace :rbs_infer do
  namespace :carrierwave do
    desc "Generate RBS files for mount_uploader/mount_uploaders, post-processing rbs_rails output"
    task :all do
      app_dir = defined?(Rails) ? Rails.root.to_s : Dir.pwd
      output_dir = File.join(app_dir, "sig/rbs_carrierwave")
      rbs_rails_dir = File.join(app_dir, "sig/rbs_rails")

      generator = RbsInfer::Extensions::CarrierWave::Generator.new(
        app_dir: app_dir,
        output_dir: output_dir,
        rbs_rails_dir: rbs_rails_dir
      )
      generator.generate_all

      puts "Generated carrierwave RBS files in sig/rbs_carrierwave/ (and stripped rbs_rails column accessors)"
    end
  end
end
```

Encadear no `Makefile`:

```make
rbs_generators_all:
	make rbs_rails_generator
	make rbs_rails_custom
	make rbs_infer_enumerize
	make rbs_infer_carrierwave    # <— novo, depois de rbs_rails_generator
	make rbs_infer_erb
```

## Snapshot tests

Adicionar em `spec/integration/rails_dummy_spec.rb`, no padrão dos existentes:

```ruby
it "CarrierWave mount_uploader rewrites User accessors" do
  require "rbs_infer/extensions/carrierwave/generator"
  Dir.mktmpdir do |tmpdir|
    # Copiar sig/rbs_rails/app/models/user.rbs pro tmpdir pra não mutar o checkout
    src_rbs_rails = File.join(tmpdir, "rbs_rails")
    FileUtils.cp_r("sig/rbs_rails", src_rbs_rails)

    generator = RbsInfer::Extensions::CarrierWave::Generator.new(
      app_dir: Dir.pwd,
      output_dir: tmpdir,
      rbs_rails_dir: src_rbs_rails
    )
    generator.generate_all

    rbs = File.read(File.join(tmpdir, "app/models/user.rbs"))
    expect(rbs.chomp).to eq(expected_rbs("carrierwave/user").chomp)

    stripped = File.read(File.join(src_rbs_rails, "app/models/user.rbs"))
    expect(stripped).not_to match(/^\s+def avatar:/)
    expect(stripped).not_to match(/^\s+def avatar=:/)
    expect(stripped).not_to match(/^\s+def avatar\?:/)
    # Mantém os *_changed?, *_was, etc.
    expect(stripped).to match(/^\s+def avatar_changed\?:/)
  end
end
```

Expectation em `spec/expectations/carrierwave/user.rbs` com o conteúdo da seção "RBS gerado" acima.
